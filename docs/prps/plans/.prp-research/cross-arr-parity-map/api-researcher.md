## Patterns to Mirror → REPOSITORY_PATTERN (contract-first API authoring)
No API-client wrapper/SDK init is needed. The reusable pattern is the repo's contract-first authoring
+ typed-handler flow:

| Category | File:Lines | Pattern | Key Snippet (≤5 lines) |
| -------- | ---------- | ------- | ---------------------- |
| Path fragment authoring | `docs/api/v1/paths/system.yaml:1-29` | File keyed by fragment name; each op has `operationId`/`summary`/`description`/`tags`/`parameters`/`responses`; response `$ref`s a schema fragment. | `health:`<br>`  get:`<br>`    operationId: getHealth`<br>`    tags: [System]`<br>`    responses: {'200': {content: {application/json: {schema: {$ref: '../schemas/health.yaml#/HealthResponse'}}}}}` |
| Query param + error responses | `docs/api/v1/openapi.yaml:45-100` (ui-preferences GET) | Query params under `parameters` (`in: query`); `400`/`401`/`404`/`500` each map to an error schema `$ref`. | `parameters:`<br>`  - name: section_key`<br>`    in: query`<br>`    required: true`<br>`    schema: {$ref: '#/components/schemas/UiSectionKey'}` |
| Schema fragment authoring | `docs/api/v1/schemas/arr.yaml:1-7,519-526` | Top-level YAML keys = schema names; plain JSON-Schema bodies. `ErrorResponse = {error: string}` is the shared error shape. | `ArrType: {type: string, enum: [radarr, sonarr, lidarr]}`<br>`ErrorResponse:`<br>`  type: object`<br>`  required: [error]`<br>`  properties: {error: {type: string}}` |
| Root registration (paths) | `docs/api/v1/openapi.yaml:36-45` | Each URL path `$ref`s a fragment: `/health: {$ref: './paths/system.yaml#/health'}` (paths may also be authored inline as ui-preferences is). | `/health:`<br>`  $ref: './paths/system.yaml#/health'` |
| Root registration (schemas) + tags | `openapi.yaml:14-34` (tags) & `openapi.yaml:614-664` (components.schemas) | Add a `tags:` entry; register EVERY response schema under `components.schemas` as `Name: {$ref: './schemas/X.yaml#/Name'}`. | `components:`<br>`  schemas:`<br>`    ErrorResponse: {$ref: './schemas/arr.yaml#/ErrorResponse'}` |

| Typed handler via contract | `routes/api/v1/arr/library/+server.ts:1-3,20-22` | Import generated `components` and alias response/error types from the contract — response types stay locked to the spec. | `import type { components } from '$api/v1.d.ts';`<br>`type LibraryResponse = components['schemas']['LibraryResponse'];`<br>`type ErrorResponse = components['schemas']['ErrorResponse'];` |
| Simple GET handler (auth + fail-fast 400) | `routes/api/v1/ui-preferences/+server.ts:30-47` | `locals.user` guard → `401`; parse query in try/catch → `400`; success via `json(...)`.
