## Gaps

- GAP: No existing `setContext`/`getContext` usage inside `$ui/form/` (only `$ui/card/` CardGrid/Card) — DisclosureSection tier integration (T11) introduces the first form-layer context consumer; validate no SSR context-timing issues.
- GAP: `clearOnAuthChange` on `userInterfacePreferencesStore` is exported but has NO caller in the codebase (grep returned no matches) — the new tier store should expose the same hook, but existing wiring for auth-change cache clearing does not yet exist.
- GAP: No existing "activity counter / progression threshold" pattern anywhere in the codebase — progression thresholds for `ComplexityProgressionHint` (T10) are net-new; threshold values are `ASSUMPTION — needs validation` by caller/product.
- GAP: Section-family vs full 3-token key granularity for tiers is undecided. Existing keys are full 3-token `route-family:page:section`; a coarser "family" (first token) grouping would require a new helper/derivation not present today. Design above reuses the full 3-token key + `SECTION_KEY_PATTERN`; family grouping left as an open decision.
